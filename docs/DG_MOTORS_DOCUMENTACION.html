🏢Introducción

DG Motors es una mecánica automotriz con dos sucursales en Ecuador (Quito y Guayaquil). Este sistema reemplaza el ERP GETSOFT con una plataforma 100% propia construida sobre Next.js 16, Supabase y TypeScript estricto.

🌐 Sitio Público
Landing, servicios, precios, sucursales, contacto. Sin autenticación.
👤 Portal Cliente
Vehículos, citas, historial de mantenimiento, facturas, perfil.
🔧 Portal Mecánico
App mobile-first: fotos de daños, checklist, estado de OTs asignadas.
⚙ Panel Admin / ERP
OTs, facturación, caja, inventario, compras, contabilidad, activos fijos.

Decisiones técnicas clave

proxy.ts (no middleware.ts)
Next.js 16 renombró middleware. El archivo se llama proxy.ts en este proyecto.
params son Promise
En Next.js 16, params y searchParams son Promise<{}>. Siempre usar await params.
Supabase SSR
createServerClient en Server Components y API Routes. createBrowserClient en Client Components.
Zod v4
Usar .issues[0].message en lugar de .errors[0].message (cambio breaking de v4).
Server Actions void
Acciones usadas en <form action={fn}> retornan Promise<void>. Las que usan useActionState retornan estado.
Sin Prisma
Queries directas con Supabase JS client. No ORM, no migraciones auto-generadas.

Flujo de autenticación

Request → proxy.ts
  → No auth  → /login (rutas /admin/*, /cliente/*, /mecanico/*)
  → Auth OK  → verificar rol en perfiles
               admin/gerente/contador/etc → /admin/*
               cliente → /cliente/*
               mecanico → /mecanico/*
               cliente en /admin/* → redirect /cliente/dashboard

🔒Seguridad y RLS

Row Level Security habilitado en todas las tablas. Los usuarios solo acceden a sus propios datos; admin ve todo.

Helpers de rol

-- is_admin() — usado en tablas pre-ERP
CREATE OR REPLACE FUNCTION is_admin()
RETURNS BOOLEAN AS $$
  SELECT EXISTS (
    SELECT 1 FROM public.perfiles
    WHERE id = auth.uid() AND rol = 'admin'
  );
$$ LANGUAGE sql SECURITY DEFINER;

-- tiene_rol(roles[]) — usado en tablas ERP/Contabilidad (migration 017+)
CREATE OR REPLACE FUNCTION tiene_rol(roles TEXT[])
RETURNS BOOLEAN AS $$
  SELECT EXISTS (
    SELECT 1 FROM public.perfiles
    WHERE id = auth.uid() AND rol = ANY(roles)
  );
$$ LANGUAGE sql SECURITY DEFINER;

📊Módulo: Contabilidad

Contabilidad en partida doble, plan de cuentas NEC/NIIF Ecuador, períodos mensuales y reportes financieros.

Principio de partida doble

Todo asiento debe tener ∑Debe = ∑Haber. La interfaz de nuevo asiento deshabilita el botón "Guardar" mientras no cuadre. Los asientos automáticos siempre cuadran por construcción.

Jerarquía del plan de cuentas

Nivel 1 — Grupo:      1 · ACTIVOS
Nivel 2 — Subgrupo:   1.1 · ACTIVO CORRIENTE
Nivel 3 — Cuenta:     1.1.01 · CAJA Y BANCOS
Nivel 4 — Subcuenta:  1.1.01.001 · Caja General  ← permite_mov = TRUE

Solo las cuentas de nivel 4 con permite_mov = true pueden tener movimientos en asiento_lineas.

Asientos automáticos

EventoMóduloAsiento generado
Emitir facturaventasDEBE Clientes / HABER Ventas + IVA
Registrar cobrocajaDEBE Caja General / HABER Clientes
Recibir compracomprasDEBE Inventario / HABER Proveedores
Depreciación mensualactivosDEBE Gasto Dep. / HABER Dep. Acumulada

Balance General

Cálculo dinámico: suma de asientos contabilizados agrupados por tipo (activo, pasivo, patrimonio). Muestra alerta si Activos ≠ Pasivos + Patrimonio.

Estado de Resultados

Filtrable por año y mes. Calcula: Ingresos − Costos = Utilidad Bruta − Gastos = Utilidad Neta. Solo asientos contabilizados del período seleccionado.

Períodos contables

Seed incluye períodos Enero–Marzo 2026 cerrados y Abril 2026 abierto. Al procesar depreciación u otros asientos automáticos, el sistema busca el período abierto correspondiente a la fecha del asiento.

Categorías disponibles (seed)

CategoríaVida útilMétodo

Cálculo de depreciación (línea recta)

depreciable   = costo_original − valor_residual
dep_mensual   = depreciable / (vida_util_anios × 12)
dep_restante  = depreciable − dep_acumulada
monto_mes     = MIN(dep_mensual, dep_restante)
← No deprecia más allá del valor depreciable

📖Manual de Usuario

Para el Administrador

Cómo crear una Orden de Trabajo
1. Ir a Admin → Órdenes → Nueva OT
2. Ingresar la placa del vehículo y hacer click en "Consultar ANT"
3. Los datos del vehículo se completan automáticamente
4. Seleccionar cliente, tipo de trabajo, km de entrada
5. Completar el checklist de recepción (22 ítems)
6. Subir fotos de daños existentes
7. Click en Crear OT
Cómo facturar una OT
1. Ir al detalle de la OT (estado: completado)
2. Click en "Generar Factura"
3. El sistema copia todas las líneas automáticamente
4. Revisar subtotal, IVA (15%) y total
5. Click en Crear Factura — se genera el asiento contable automáticamente
6. Desde la factura, click en "Registrar Pago" para cobrar
Cómo procesar la depreciación mensual
1. Ir a Admin → Activos Fijos → Depreciación
2. Seleccionar el año y mes a procesar
3. Click en "Procesar Depreciación"
4. El sistema calcula la cuota mensual de cada activo activo
5. Actualiza la depreciación acumulada de cada activo
6. Genera automáticamente el asiento contable
7. Solo se puede procesar una vez por período
Cómo ver el Balance General
Ir a Admin → Contabilidad → Balance General
Muestra el balance al día de hoy con todos los asientos contabilizados.
Si hay diferencia entre Activos y Pasivos+Patrimonio, aparece una advertencia en amarillo.

Para el Cliente

Cómo registrar un vehículo
1. Iniciar sesión en /login con sus credenciales
2. Ir a Mis Vehículos → Registrar Vehículo
3. Ingresar la placa y hacer click en "Consultar ANT" para auto-completar
4. Verificar o completar los datos restantes
5. Click en Guardar Vehículo
Cómo ver el historial de mi vehículo
1. Ir a Mis Vehículos
2. Click en el vehículo deseado
3. Ver historial completo de mantenimientos, cambios de aceite y OTs
4. También disponible en Mantenimiento para ver todo en un timeline

Para el Contador

Acceso al módulo contable
El contador accede a todo desde Admin → Contabilidad:
• Plan de Cuentas — árbol jerárquico NEC/NIIF
• Asientos — todos los asientos (manuales y automáticos)
• Períodos — abrir y cerrar períodos mensuales
• Balance General — activos, pasivos, patrimonio
• Estado de Resultados — ingresos, costos, gastos, utilidad
• Activos Fijos — registro y depreciación de activos

Setup inicial del proyecto

# 1. Clonar repositorio
git clone https://github.com/kevalty/dgmotors.git
cd dgmotors

# 2. Instalar dependencias
npm install

# 3. Crear .env.local (copiar de .env.local.example y completar)
cp .env.local.example .env.local

# 4. Ejecutar migrations en Supabase SQL Editor (en orden)
# supabase/migrations/001_schema_inicial.sql
# ... hasta 020_ventas_avanzado.sql (ejecutar en orden: 001 → 020)

# 5. Iniciar servidor de desarrollo
npm run dev

# 6. Acceder a http://localhost:3000
