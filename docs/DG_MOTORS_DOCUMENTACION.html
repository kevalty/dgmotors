🎯 Visión General

DG Motors es una mecánica automotriz con dos sucursales en Ecuador (Quito y Guayaquil), especializada en vehículos americanos, Ford y multimarca. Esta plataforma reemplaza su ERP anterior (GETSOFT) con un sistema 100% propio.

🌐 Sitio Público
Landing, servicios, precios, sucursales con mapas Google, formulario de contacto y catálogo de productos.
👤 Portal Cliente
Gestión de vehículos, agendamiento de citas, historial de mantenimiento, cambios de aceite y facturas.
⚙ Panel Administrativo
Gestión completa: OTs, inventario, facturación, caja, compras, proveedores, reportes y contabilidad.
🔧 Portal Mecánico
App móvil simplificada: citas asignadas, fotos de daños, checklist de recepción y registro de trabajo.
📊 Módulos ERP
Órdenes de trabajo, inventario por bodegas, facturación con IVA 15%, caja diaria y compras a proveedores.
📒 Contabilidad
Plan de cuentas NEC/NIIF Ecuador, asientos de partida doble, períodos contables y centros de costos.

🚀 Instalación y Setup

Requisitos

  • Node.js 18+ / npm 9+
  • Cuenta en Supabase (proyecto PostgreSQL)
  • Variables de entorno configuradas (ver sección Variables de Entorno)

Instalación

# 1. Clonar repositorio
git clone https://github.com/kevalty/dgmotors.git
cd dgmotors

# 2. Instalar dependencias
npm install

# 3. Instalar browsers de Playwright
npx playwright install chromium

# 4. Configurar variables de entorno
cp .env.local.example .env.local
# → Editar .env.local con tus credenciales Supabase

# 5. Ejecutar migrations en Supabase SQL Editor (en orden)
#    supabase/migrations/001_schema_inicial.sql
#    ... (hasta 017_contabilidad.sql)

# 6. Iniciar servidor de desarrollo
npm run dev

Fotos de Entrada / Salida

Las fotos se suben al bucket ot-fotos de Supabase Storage. Soporta múltiples fotos por OT, organizadas por tipo (entrada / salida). Visibles en la pestaña "Fotos" del detalle de OT.

Email Automático

Al cambiar el estado de una OT, el sistema envía un email al cliente via Resend con el nuevo estado, descripción del trabajo y link al portal del cliente. Requiere RESEND_API_KEY en las variables de entorno.

Asientos Contables

El sistema valida que ∑ Debe = ∑ Haber antes de permitir guardar el asiento. El botón "Guardar Asiento" permanece deshabilitado hasta que el asiento cuadre.

Estados del asiento: borrador → contabilizado (no editable) o anulado.

Períodos Contables

Seed 2026: enero–marzo cerrados, abril abierto. Los asientos solo pueden crearse en períodos abiertos. El cierre es irreversible salvo apertura manual por el contador.

🚗 Integración ANT Ecuador

Feature estrella del demo. Al ingresar una placa ecuatoriana, el sistema consulta la ANT y autocompleta todos los datos del vehículo automáticamente.

Funcionamiento técnico

// lib/ant/consultar.ts
async function consultarPlacaANT(placa: string) {
  // 1. Verificar caché en tabla ant_cache (30 días de TTL)
  const cache = await supabase
    .from('ant_cache')
    .select('*')
    .eq('placa', placa)
    .gt('expira', new Date().toISOString())
    .maybeSingle()

  if (cache.data) return cache.data.datos  // hit de caché

  // 2. Consultar API externa ANT
  const res = await fetch(
    `${ANT_API_URL}?placa=${placa}&apikey=${ANT_API_KEY}`
  )

  // 3. Guardar en caché y retornar datos
}
⚠ Configuración requerida: Se necesita una ANT_API_KEY válida de placaapi.ec para que la consulta funcione con datos reales. En desarrollo/demo, el mock de Playwright intercepta la llamada.

🛡 Seguridad y RLS

Row Level Security habilitado en todas las tablas. Los usuarios solo pueden ver y modificar sus propios datos.

ReglaDetalle
PerfilesUsuario solo ve su propio perfil. Admin ve todos.
VehículosCliente solo ve sus vehículos (WHERE cliente_id = auth.uid())
CitasCliente solo ve sus citas. Admin ve y modifica todas.
MantenimientosCliente solo lectura de sus vehículos. Admin full CRUD.
Facturas / OTsAdmin y roles ERP. Cliente solo sus facturas en /cliente/mis-facturas.
ContabilidadSolo admin y contador pueden escribir. Gerente solo lectura.
Inventario / ComprasSolo admin, asesor_repuestos y bodeguero.
CajaSolo admin y facturadora pueden operar la caja.

Función helper de rol

-- Usada en todas las políticas RLS
CREATE OR REPLACE FUNCTION tiene_rol(roles TEXT[])
RETURNS BOOLEAN AS $$
  SELECT EXISTS (
    SELECT 1 FROM public.perfiles
    WHERE id = auth.uid() AND rol = ANY(roles)
  );
$$ LANGUAGE sql SECURITY DEFINER;
⚠ Regla de oro: La verificación de permisos se hace siempre en el servidor (Server Components, Server Actions, API Routes). Nunca confiar en validaciones solo del cliente.
